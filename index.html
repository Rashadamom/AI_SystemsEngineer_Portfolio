// The Problem

IT staffing firms spend 3–4 weeks screening, submitting, and interviewing candidates who then ghost or drop out at offer stage. The signals — inflated salary demands, 90-day notice periods, no resume ready, days of silence — were always present. But buried in spreadsheets with 40 other rows, no one acted on them in time. The cost is wasted recruiter hours, lost client trust, and missed placements.

// What Was Built
  • Google Sheets trigger ingests new candidate entries in real-time
  • Normalization layer standardizes all intake fields for consistent scoring
  • Validation and enrichment gate catches bad data before it enters the pipeline
  • Custom JavaScript scoring engine computes dual-score rubric (ghosting probability + commitment index)
  • Conditional router: high-risk profiles auto-escalate to human review queue
  • Candidate master tracker and stage update log maintained with full history
  • Invalid intake routed to error handler — no silent failures
// Workflow Pipeline
CANDIDATE_INTAKE→ NORMALIZE→ VALIDATE_ENRICH→ VALIDATION_OK?→ SCORE_RISK→ REVIEW_REQUIRED?→ UPSERT_MASTER / REVIEW_QUEUE

// The Problem

Real estate agents spend $1,000+/month on lead generation from platforms like CINC and Ylopo but convert only 2–3% of those leads. The data needed to identify buyer intent, predict ghosting risk, and personalize outreach already exists — it just isn't connected, enriched, or analyzed at intake. Agents follow up with every lead the same way, with the same message, regardless of signal strength.

// What Was Built
  • Landbot chatbot webhook captures buyer/seller/renter intent at point of contact
  • GPT-4 LangChain Behavioral AI Agent profiles each lead's intent and behavioral signals
  • BatchData API pulls live property and ownership data to enrich lead context
  • Market Intelligence Processor generates personalized outreach per lead type and location
  • Smart Agent Routing assigns leads to the right realtor based on client config
  • Twilio SMS with full opt-out compliance logic — no regulatory exposure
  • Salesforce CRM sync: leads, agents, activities, property search records
  • Google Sheets audit layer — full pipeline logging for every action
// 4-Workflow Architecture
WF1: Agent Onboarding→Webhook → Process each client/agent → Upsert to Sheets + Salesforce
WF2 Pt1: Lead Intake→Landbot → Behavioral AI Agent → Parse + validate → Opt-out check → Log migration
WF2 Pt2: Market Intel→BatchData property API → Enrich → Smart agent routing → Upsert Lead to Salesforce
WF2 Pt3: Comms + Log→Generate SMS with market intel → Twilio → Compliance check → Log to SF + Sheets

// The Problem

Grammarly checks grammar. Linters check code. Nothing checks requirements for ambiguity. "Checkout should be fast" means 3 seconds to one team, 500ms to another — 3 sprints wasted. "Administer medication as needed" with no dosage thresholds is a patient safety risk. After 12 years seeing this firsthand in enterprise healthcare IT, I built the tool I always wished existed.

// What Was Built
  • 3-agent pipeline: Agent 1 detects vague language and passive constructions, Agent 2 scores 5 dimensions with cited evidence, Agent 3 rewrites with measurable acceptance criteria
  • Temperature 0.0 — deterministic and auditable, not a black box
  • Kanban board UI — tickets auto-route by risk score (Red / Yellow / Green)
  • Batch Scan: paste entire sprint backlog, get full risk picture in 60 seconds
  • API-first: POST /api/analyze returns structured JSON for Jira, Azure DevOps, or any system
  • On-premise deployment path for government and HIPAA-strict enterprise clients

// The Problem

Enterprise and government clients need AI-powered analysis but cannot send sensitive operational documents to cloud APIs. HIPAA, ITAR, and data sovereignty mandates make cloud LLMs a non-starter. This is a gap most automation engineers cannot fill — it requires both AI architecture knowledge and an understanding of regulated-industry constraints.

// What Was Built
  • Runs 100% offline — no data leaves the local machine at any point
  • LM Studio backend serves a local LLM via OpenAI-compatible API endpoint
  • Next.js + shadcn/ui + Tailwind frontend with Jira-style Kanban workflow UX
  • Analyzes operational documents for risk, ambiguity, and compliance gaps
  • Designed for regulated environments: healthcare, government, defense
  • Proof-of-concept deployable for enterprise clients with strict data residency requirements
